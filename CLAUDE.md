## Project Overview

**SoyYo / Garantir** — Flutter app (Android + Web) for product warranty management. Users register products with purchase and expiry dates, associate them with categories, attach invoice photos (camera), and receive local push notifications before warranties expire.

## Commands

```bash

flutter pub get

# Generate Isar schema code (run after any change to *_isar_schema.dart files)
flutter pub run build_runner build --delete-conflicting-outputs

# Run on Android
flutter run -d android

# Run on Web (Chrome)
flutter run -d chrome

# Analyze code (lints)

# Run tests
```

## Data Layer: Isar + Firebase

**Read path:** All `watch*()` methods in `ProductRepository` return **Isar streams** — fast, local, offline-capable.

**Write path:** Every save/delete writes to **Isar first** (immediate UI update), then calls Firestore fire-and-forget (no `await`).

**Sync periódico:** `ProductRepository.syncIfNeeded()` é chamado no `initState` do `ProductsPage`. Verifica o timestamp `lastSyncAt` em `SyncMetaIsarSchema` (registro único, id=1). Se nunca sincronizou ou se passaram ≥14 dias, faz um `get()` único no Firestore e atualiza o Isar. Não usa listeners contínuos — o app funciona 100% offline entre syncs.

**Isar IDs:** Isar exige `int` IDs. Firebase usa UUID strings. A função `fastHash()` (FNV-1a 64-bit) converte o UUID em um `int` estável usado como ID do documento. O UUID original é preservado em `firestoreId` (índice único).

**No state management:** Services and repositories are instantiated directly in widgets. Riverpod/Provider is a planned next step.

## Login persistente

Firebase Auth mantém a sessão salva localmente (IndexedDB no Web, armazenamento nativo no Android) — o usuário permanece logado entre aberturas do app até fazer logout explícito. Nenhuma implementação extra necessária.

## Auth Flow

`app.dart` usa `StreamBuilder<User?>` em `FirebaseAuth.instance.authStateChanges()`:
- `null` (não autenticado) → `LoginPage`
- `User` (autenticado) → `ProductsPage`

A navegação entre as telas é automática — não há `Navigator.push` explícito após login/registro. O logout (`AuthService.signOut()`) limpa o estado e o StreamBuilder redireciona para `LoginPage`.

> **Nota:** os dados no Firestore ainda não são isolados por usuário. O `currentUser.uid` deve ser usado como prefixo nas coleções (`users/{uid}/products`) em uma próxima iteração.

## Firebase Setup

`lib/firebase_options.dart` é um placeholder com strings dummy. Antes de executar o app:

```bash
dart pub global activate flutterfire_cli
flutterfire configure   # sobrescreve lib/firebase_options.dart com as credenciais reais
```

Serviços Firebase necessários: **Authentication** (Email/Password), **Firestore Database**, **Firebase Storage**.

## Firestore Schema

**`products`** collection: `name` (string), `categoryId` (string), `purchaseDate` (timestamp), `warrantyEndDate` (timestamp), `guaranteeType` (string — enum name), `noteImageUrl` (string?), `description` (string?)

**`categories`** collection: `name` (string), `isDefault` (bool)

Three default categories are seeded on first run via `seedDefaultCategories()` using fixed IDs — the method checks Isar before writing to avoid duplicates.

## Notifications

`WarrantyAlertService` schedules two local notifications per product: 7 days before expiry and on the expiry day (both at 09:00). Timezone is hardcoded to `America/Sao_Paulo`. Silently skipped on Web (`kIsWeb` guard). Notification IDs are derived from a hash of the product UUID.
